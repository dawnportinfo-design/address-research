# Address UX Standard

Address UX should reduce user burden while keeping private address material hidden by default.

## Research Tasks

1. Define input states: verified, partial, manual required, ambiguous, unresolved.
2. Design autocomplete and correction flows that preserve user trust.
3. Separate user-facing clarity from internal validation complexity.
4. Measure cognitive load and error recovery.

## Safety Boundary

Demos should prefer alias, receipt, AGID, AOID, commitment, and quality status over raw address text.
