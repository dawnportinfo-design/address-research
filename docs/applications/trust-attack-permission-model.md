# Trust, Attack, and Permission Model

Address infrastructure is high-risk because addresses connect people, places, movement, identity, payments, and logistics.

## Research Tasks

1. Define attacker classes: scraper, dishonest integrator, compromised carrier, replay attacker, malicious verifier, and insider.
2. Define permission scopes for registration, validation, translation, proof, handoff, and revocation.
3. Require audit logs that do not store private payloads.
4. Define publication gates for datasets and examples.

## Safety Boundary

No secrets, credentials, private keys, witnesses, decrypted QR payloads, or recipient data.
