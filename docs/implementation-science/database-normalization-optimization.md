# Database Normalization and Optimization

Address databases should separate referents, labels, evidence, history, aliases, quality, and privacy scopes.

## Research Tasks

1. Define normalized schemas for country models, evidence, history, and identifiers.
2. Compare relational, graph, spatial, and event-sourced designs.
3. Specify indexes for autocomplete, validation, and reverse lookup.
4. Analyze update and query complexity.

## Safety Boundary

No private production datasets or reconstructable raw address fixtures.
